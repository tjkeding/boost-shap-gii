      <description>

      </description>

    <finding id="F24" severity="note" category="validity">
      <location file="shap_utils.py" lines="369,384" />
      <description>
        The SHAP decomposition identity is: SHAP_total(x_i) = Φ(i,i) + Σ_{j≠i} Φ(i,j). The pipeline extracts singletons as Φ(i,i) and interactions as Φ(i,j) + Φ(j,i). For a symmetric interaction matrix, this means the total attribution for feature i is: Φ(i,i) + Σ_{j≠i} [Φ(i,j) + Φ(j,i)] = Φ(i,i) + 2·Σ_{j≠i} Φ(i,j). This is 2× the original SHAP_total minus Φ(i,i). The doubling of off-diagonal terms is consistent with the SHAP interaction decomposition convention where the total effect includes both directions of the interaction, but users should be aware that summing singleton + all interaction GIIs does not reconstruct the marginal SHAP importance.
      </description>
      <evidence>shap_utils.py:369 — singleton = Φ(i,i); shap_utils.py:384 — interaction = Φ(i,j) + Φ(j,i).</evidence>
      <impact>Interpretive. No computational error. Users attempting to "reconstruct" marginal importance from the GII table will get incorrect results.</impact>
      <recommendation>Document in README that GII values represent decomposed effects and cannot be naively summed to reconstruct marginal SHAP values.</recommendation>
    </finding>

    <finding id="F25" severity="note" category="generalizability">
      <location file="shap_utils.py" lines="206-239" />
      <description>
        The V component uses the standard deviation of the fitted spline signal as a measure of structured variability. This assumes that a smooth trend in SHAP-vs-feature space represents genuine systematic variation. For highly non-linear relationships (e.g., U-shaped SHAP curves), the spline captures this well. However, for features where the SHAP-vs-feature relationship is genuinely noisy (high individual-level variation around a weak trend), the spline may overfit to noise in some bootstrap resamples and underfit in others. The energy gate mitigates overfitting but does not address underfitting.

        The GII = sqrt(M * V) formulation means that a feature with large M (strong average effect) but low V (no systematic trend) will have low GII. This is by design — it captures the insight that a feature can have large SHAP magnitudes that are randomly distributed across feature values (no structured dependency). However, this also means the GII framework is insensitive to features whose importance is "level" — constant across all feature values but consistently non-zero.
      </description>
      <evidence>shap_utils.py:237 — `return np.std(signal)` where signal is the spline fit.</evidence>
      <impact>Features with constant (non-zero) SHAP values across all feature values will have V≈0 and GII≈0. These features have genuine predictive importance (high M) but no dose-response relationship. The pipeline correctly identifies them as having no structured variability, but users may be confused that a feature with high marginal SHAP importance has GII≈0.</impact>
      <recommendation>Document this property clearly. Consider reporting M as a separate importance metric alongside GII for features where V≈0 but M is large, so users can identify "constant-effect" features.</recommendation>
    </finding>
